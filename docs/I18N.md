# NutriSync — Internationalization (Spanish, es-419)

**Status:** live. Users can choose **Español (Latinoamérica)** at sign-up or in
Settings → Preferences → Language. English remains the default.

## Architecture

Zero-dependency, typed dictionary layer — no route restructuring, no library:

| Piece | File | Role |
|---|---|---|
| Dictionaries | `lib/i18n/dictionaries.ts` | `en` is the schema; `es: Dict = typeof en` makes a missing/extra Spanish key a **compile error**. Parameterized strings are functions (`stepOf(n, total)`), so plurals/interpolation stay type-safe. |
| Helpers | `lib/i18n/index.ts` | `Locale`, `isLocale`, `resolveLocale`, `getDict`, `htmlLang` (`es` → **es-419**), `LOCALE_COOKIE`. |
| Server | `lib/i18n/server.ts` | `getLocale()` from the `nutrisync_locale` cookie. |
| Client | `components/I18nProvider.tsx` | Context mounted in the root layout with the server-read locale (no hydration flash); `useI18n()` → `{ t, locale }`; `setLocaleCookie()`. |
| Persistence | `supabase/migrations/051_profile_language.sql` | `profiles.language ('en'\|'es')` — durable, cross-device. |

**Preference flow:** the cookie is the per-device value (works pre-auth, so the
login page itself renders Spanish); the profile column is the account truth.
Sign-up copies the chosen language cookie → profile; sign-in copies profile →
cookie (or adopts the device choice if the account has none yet); the Settings
toggle writes both. The root layout reads the cookie per request — this makes
all routes dynamic, which the core routes already were.

## Adding a string

1. Add the key to `en` in `lib/i18n/dictionaries.ts` (pick the right section).
2. TypeScript now errors until you add the matching `es` entry — add it
   (LatAm conventions: **tú** form, `correo`, `contraseña`, `configuración`,
   `racha`, `meta`).
3. In a client component: `const { t } = useI18n()` → `t.section.key`.
   In a server component: `const t = getDict(await getLocale())`.
4. `tests/i18n.test.ts` walks both dictionaries (empty strings fail CI).

## Adding a locale

Add the code to `Locale`/`LOCALES`, a dictionary in `dictionaries.ts` (the
`Dict` type enforces completeness), a row in the Settings language page and the
login picker, and extend the `profiles_language_check` constraint.
